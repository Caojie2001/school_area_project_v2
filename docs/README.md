## 📖 文档规范

### 文档命名
- 使用大写字母和下划线: `PROJECT_STRUCTURE.md`
- 文件名简洁明了，体现内容主题
- 避免使用空格，使用下划线或连字符

### 文档结构
- 标题清晰，层级分明
- 使用 emoji 增强可读性（可选）
- 代码块标注语言类型
- 提供目录导航（对于长文档）

### 维护原则
- 代码变更时同步更新文档
- 及时移除过时内容
- 保持示例代码的准确性
- 添加更新日期和版本号
